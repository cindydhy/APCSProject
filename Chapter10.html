Chapter 10: ArrayLists


ArrayLists

  • Internally, each ArrayList object uses an array to store its values. As a result, an ArrayList provides the same fast random access as an array. But unlike with an array, with an ArrayList you can make simple requests to add or remove values, and the ArrayList takes care of all of the details for you: If you add values to the list it makes the array bigger, and if you remove values it handles any shifting that needs to be done.
  • In the API documentation for ArrayList, the syntax is actually listed as ArrayList(E). which is an example of a generic class in Java. It takes a type parameter to indicate what kind of values it will use.
    The “E” is short for “Element,” and it indicates the type of elements that will be included in the ArrayList. Generic classes are similar to parameterized methods.


  • The E in ArrayList can be filled in with any object or reference type. The primitive types cannot be used as type parameters for an ArrayList. Instead, Java defines a series of wrapper classes that allow you to store primitive data as objects.
  • Boxing is the automatic conversion from primitive data to a wrapped object of the appropriate type
  • Unboxing is the automatic conversion from a wrapped object to its corresponding primitive data.

The Comparable Interface

  • Natural Ordering is the order imposed on a type by its comparison function
  • Java has a convention for indicating the natural ordering of a type. Any type that has such an ordering should implement the Comparable interface:
    public interface Comparable(T) {
    public int compareTo(T other);
    }
  • The convention for compareTo is that an object should return one of the following results:
    A negative number to indicate a less-than relationship
    0 to indicate equality
    A positive number to indicate a greater-than relationship
  • We have to replace the (T) in Comparable(T). Whenever you implement Comparable, you compare pairs of values from the same class.
