Chapter 3: Introduction To Parameters and Objects


Parameters

  • Parameter (Parameterize): any set of characteristics that distinguish different members of a family of tasks. To parameterize a task it to identity a set of its parameters.
  • Format of passing parameters:
    • Ex.: public static void methodName(type parameter){}
  • Method call:   methodName(type name);
  • Formal parameter: variable that appears inside parentheses in the header of a method that is used to generalize the method's behavior
    • Ex.: public static void name(int a)
  • Actual parameter: specific value or expression that appears inside parentheses in a method call
    • Ex.: name(9);
  • When a parameter is set up, a local variable is created and initialized to the value of the actual parameter. Since it is a local variable, it can't influence any variables outside the method
    • See pg144 for an example

Different types of methods

  • Method signature: name of a method, along with its number and type of parameters
    • Ex.: public static void name(String word)
  • Overloading methods: ability to define two or more methods with same name but different method signatures
    • Acceptable method signatures
      • p.s.v. name(int a, double b)
      • p.s.v. name(String a)
      • p.s.v. name(double a, int b)
  • Methods that return values
    • Return: send a value out as the result of a method that can be used in an expression
      • Void methods do not return any value
    • Ex.: public static double sqrt(double n)
      • double is the return type

Classes and objects

  • Class: program entity that represents either a program or a type of object
    • Program: your HelloWorld program
    • Object: Math class
  • Object: entity that combines data and behavior
    • Object-oriented programming (OOP): programs that perform their behavior as interactions between objects
    • Data: variables inside the object (i.e. values, strings)
    • Behavior: methods inside the object
  • Constructing an object:
    • Type objectName = new Type(parameters);
    • Ex.: Scanner console = new Scanner(System.in);
  • Calling an object's method
    • objectName.methodName(parameters);
    • Ex.: console.hasNext();
  • Package: collection of related Java classes
  • Import declaration: request to access a specific Java package

Strings

  • Characters of a string numbered with 0-based indices
  • First characer's index:   0
  • Exception: runtime error that prevents a program from continuing its normal execution
  • Immutable object: an object whose balue cannot be changed
