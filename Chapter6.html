Chapter 6: File Processing


File-Reading Basics

  • Files can be text files or binary files depending on the format.
    Text files can be edited using text editors. Binary files are stored using an internal format that requires special software to process.




  • You construct a File object by the line of code:
    File f = new File("hamlet.txt");
  • Once the object is constructed, the following methods will be called to manipulate the file.




  • The following line of code instructs the computer to construct a Scanner that reads from the console
    Scanner console = new Scanner(System.in);
  • Checked Exception is an exception that must be caught or specifically declared in the header of the method that might generate it

Details of Token-Based Processing

  • Token based processing is processing input token by token (i.e., one word at a time or one number at a time)
  • For the computer, each file is just a one-dimensional sequence of characters
    This sequence is how the computer views the file: as a one-dimensional sequence of characters including special characters that represent “new line”.
    308.2 14.9 7.4\n2.8\n\n\n3.9 4.7 -15.4\n2.8\n
  • File Path is a description of a file’s location on a computer, starting with a drive and including the path from the root directory to the directory where the file is stored
  • Current Directory (Working Directory) is the directory that Java uses as the default when a program uses a simple file name.

Line-Based Processing

  • Line-Based Processing is the practice of processing input line by line (i.e., reading in entire lines of input at a time)
    You can use the nextLine and hasNextLine methods of the Scanner object
    Scanner input = new Scanner( new File("poem.txt"));
    while (input.hasNextLine()) {
    String text = input.nextLine();
  • When a file requires a combination of line-based and token-based processing, you can construct a String-based Scanner for each line of the input file
  • if you have a file composed of n lines of input, you end up constructing n + 1 different Scanner objects–one for each of the n individual lines and one extra Scanner that is used to process the overall file line by line.

Advanced File Processing

  • System.out is a variable that stores a reference to an object of type PrintStream. The print and println statements are part of the PrintStream class. The variable System.out stores a reference to a special PrintStream object that is tied to the console window. However, you can construct other PrintStream objects that send their output to other places.
    For example, if you want to send output to a file called results.txt. You can construct a PrintStream object as follows:
    PrintStream output = new PrintStream(new File("results.txt"));
  • The File class has a method called canRead that you can use to test whether a file exists and can be read. You can print an error message each time the file can’t be read, until you get a good file name.
