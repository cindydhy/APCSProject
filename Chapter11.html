Chapter 11: Java Collections Framework


Definitions

  • Collection: an object that stores a group of other objects, called its elements
  • List: ordered collection of elements, often accessed by integer indexes or by iteration
  • Stack: collection in which the last element added is the first one to be removed
  • Queue: collection in which elements are removed in the same order in which they were added
  • Set: collection of elements that is guaranteed to contain no duplicates
  • Map: collection of key/value pairs in which each key is associated with a corresponding value
  • keySet: a Set of all keys in the map
  • value: a collection of all values (data) in the map



Linked List

  • Nodes: small individual containers in which elements of a linked list are stored
  • Linked list: collection that stores a list of elements in nodes which are linked together

Iterator

  • iterator: object that allows a client to traverse the elements of any collection

Abstract Data Type

  • Abstract Data Type (ADT): specification of a type of data and the operations that can be performed on it

Sets

  • HashSet
    • Extremely fast performance for add, contains, and remove tasks
    • Can be used for any type of objects as its elements
  • TreeSet
    • Elements stored in sorted order
    • Must be used with elements that can be compared (i.e. integers, strings)
