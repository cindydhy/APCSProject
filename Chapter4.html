Chapter 4: Conditional Execution


If/else statements

  • A conditional is something that executes only when the statement is sometimes true
  • If it is true then the next block of statements will execute. If false, then the block of statements is skipped.
  • There are more than two parts: if and else. However, remember that there are three parts that make up this conditional statement: if, else if, and else.

Cumulative algorithms

  • A cumulative algorithm is an operation in which an overall value is computed incrementally, often using a loop
  • Cumulative sums help increase a sum by a certain variable
    For example: sum = sum + next or sum += next;
  • If statements can also be used in cumulative statements.
  • Roundoff errors occur when floating-point numbers are used because they are stored as approximations rather than exact values.
    Using “import java.math.BigDecimal” will help solve the error because the BigDecimal class can handle very large and very small floating point numbers with great precision but it would take a little longer to compute and run
    BigDecimal premium = BigDecimal.valueOf(1586.6d);

Text processing

  • It is the editing and processing of strings of text.
  • The char type is a primitive type which represents a single character of a text.
  • It uses up small amounts of memory and doesn’t have any methods.
    For example: char ch = 'A';
  • The characters of a String are stored inside the object as values of type char.
  • Char values are stored as 16 bit integers and Java automatically converts char to int when it’s expecting an int.
    They can be compared by using relational operators (< or ==)

Methods with conditional execution

  • Throwing exceptions
    Since exceptions are for objects, before you throw an exception, you must construct an exception object using new.
    The exception object has information about what happened when the error occurred. There’s a class called IllegalArgumentException that covers cases where someone has passed an inappropriate value as an argument.
