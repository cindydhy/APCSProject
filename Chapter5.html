Chapter 5: Program Logic and Indefinite Loops


The while loop

  • Definite loop: executes known number of times
  • Indefinite loop: number of times its body repeats not known in advance
  • while loop: repeatedly executes its body as long as logical test is true
    • while loop syntax
      • while (test){
             statement(s);
        }
  • Pseudorandom numbers: numbers that, although they are derived from predictable and well-defined algorithms, mimic the properties of numbers chosen at random
    • Ex.: Math.random();
      • Between 0.0 <= x < 1.0
    • Ex.: Random class in java.util package
      • Random r = new Random();



  • Priming a loop: initializing variables before a loop to "prime the pump" and guarantee that is entered

Fencepost Algorithms

  • Fencepost algorithms: before or after a loop an operation/function is executed
    • See pg 324
  • Sentinel: special value that signals end of input

The boolean type



  • Short-circuited evaluation: property of the logical operators && and || that prevents the second operand from being evaluated if the overall result is obvious from the value of the first operand

User errors

  • Robust: ability of a program to execute even when presented with illegal data
    • Generate an error message when illegal data is present

Assertions and program logic

  • Assertion: declarative sentence that is either true or false
  • provable assertion: assertion that can be proven to be true at a particular point in program execution
  • Formal verification: field of computer science that involves reasoning about formal properties of programs to prove the correctness of a program
